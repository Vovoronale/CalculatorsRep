# ЄДЕССБ та ПД: структура розділу

## Мета

Перетворити розділ «Норми та перевірки» на компактний каталог інструментів, пов'язаних з ЄДЕССБ і підготовкою проєктної документації, без дублювання профільних інженерних калькуляторів.

## Погоджена структура

- Батьківський розділ отримує назву «ЄДЕССБ та ПД».
- Поточний slug `normy-perevirky` зберігається, щоб не змінювати наявні внутрішні посилання.
- У розділі залишаються дочірні категорії «Нормоконтроль» і «Клас наслідків».
- Категорія «DXF / GeoJSON» переноситься до «ЄДЕССБ та ПД» разом із калькулятором IV GeoJSON.
- Батьківська категорія «CAD / GIS / Дані» видаляється.
- Порожні після очищення категорії «Перевірки ДБН» і «Нормативні обґрунтування» видаляються.

## Усунення дублів

З `extraCategories` прибираються посилання на старий розділ та його видалені підкатегорії для таких калькуляторів:

- «Розрахунковий опір ґрунту основи»;
- «Напруження під підошвою фундаменту»;
- «Категорії та групи сталевих конструкцій»;
- «Клас впливу середовища для бетону»;
- «Захисний шар бетону для арматури».

Ці калькулятори залишаються тільки у своїх основних профільних категоріях.

## Супровідні зміни

- Опис та іконка батьківського розділу мають відповідати темі ЄДЕССБ і проєктної документації.
- `CategorySlug` оновлюється відповідно до видалених категорій; slug `dxf-geojson` зберігається.
- Інших категорій, калькуляторів або видимого контенту зміна не стосується.

## Перевірка

- `npm run typecheck`
- `npm run build`
- Візуальна перевірка структури категорій і відсутності дублів на головній сторінці.
