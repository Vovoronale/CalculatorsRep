## Agreed Scope

The calculator determines areas of the agreed platform types:

```text
Дитячі майданчики.
Майданчики для відпочинку дорослого населення.
Майданчики для занять фізкультурою.
Відкрита гостьова стоянка автомобілів.
Майданчики для тимчасової стоянки велосипедів.
Майданчики для збирання побутових відходів.
Майданчики для вигулу домашніх тварин.
Умовні господарські майданчики.
```

The calculator does not determine the minimum residential land parcel under
table 6.3 and does not verify actual placement, landscaping, insolation, access
roads, fire requirements, sanitary distances, local planning restrictions, or
technical conditions.

## Calculation Convention

The calculator always receives both the number of residents and the apartment
composition. For every platform type that has both table 6.4 rates, it calculates
both areas and adopts the larger value:

```text
S = max(Sос; Sкв)
```

This is a conservative calculator rule, not a separate DBN requirement.

## Units And Numeric Formatting

- The calculation core normalizes all area inputs to `м²`, `м²/особу`, or
  `м²/квартиру` before applying the canonical formulas.
- Supported user input units are `м²`, `ар`, and `га`, including their
  per-person and per-apartment forms.
- Calculations and totals use full-precision normalized values. Intermediate
  display values are never rounded before addition.
- UI, report, and DOCX area values show no more than two decimal places and
  omit unnecessary trailing zeroes.
- Guest parking spaces are integers and use ceiling only in the canonical
  `ceil` formula.
- In the formulas below, placeholders such as `<Sвідх,руч>`, `<qгосп,ос>`,
  and `<qгосп,кв>` mean normalized base-unit values.

For a dimensioned manual input in the base unit, show:

```text
Sвідх,руч = <input> м²
qгосп,ос = <input> м²/особу
qгосп,кв = <input> м²/квартиру
```

For a dimensioned manual input in another unit, show the conversion:

```text
Sвідх,руч = <input> <selected_area_unit> = <normalized> м²
qгосп,ос = <input> <selected_area_unit>/особу = <normalized> м²/особу
qгосп,кв = <input> <selected_area_unit>/квартиру = <normalized> м²/квартиру
```

## Validation And Stable Failure Behavior

Errors:

```text
Кількість мешканців має бути цілим числом, більшим за 0.
Кількість однокімнатних квартир має бути цілим числом, не меншим за 0.
Кількість дво- та більше кімнатних квартир має бути цілим числом, не меншим за 0.
Загальна кількість квартир має бути більшою за 0.
Зменшений норматив не можна застосувати без окремої озелененої зони з фізкультурними майданчиками.
Зменшений норматив не можна застосувати без забезпечення не менше 6 м² зелених насаджень обмеженого користування на одну особу.
Для вакуумного способу введіть площу майданчика за технічними умовами, більшу за 0 м².
Питомий розмір господарського майданчика на одну особу має бути від 0,1 до 0,3 м²/особу.
Питомий розмір господарського майданчика на одну квартиру має бути від 0,25 до 0,75 м²/квартиру.
```

Rules:

- All numeric values must be finite.
- Conditional-field errors are evaluated only while the field is displayed.
- Any error sets `valid = false`.
- Invalid reports retain the entered input, the 12-step order, and every result
  that can still be calculated safely.
- Formula strings must never contain `NaN`, `Infinity`, or empty numeric
  substitutions.
- The calculator does not create an automatic error from the resident-to-
  apartment ratio because the DBN does not define one universal boundary.
